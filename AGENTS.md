\# 🔷 Project Context (IMPORTANT - اقرأ قبل أي رد)



أنت تعمل كمطور داخل مشروع اسمه \*\*Heavy System\*\*

نظام SaaS لإدارة الأجهزة + المشتركين + الدعم + الإشعارات



\---



\## 🎯 هدفك في هذا الشات



\* تكمل على نفس المشروع (NOT from scratch)

\* تحافظ على نفس أسلوب الكود والتنظيم

\* أي تعديل يكون "تحسين" وليس تخريب أو إعادة بناء



\---



\## 🧠 ملخص المشروع



\* Backend: Python (Flask - Blueprints جاري تقسيمها)

\* فيه modules:



&#x20; \* support (نظام الدعم)

&#x20; \* devices (إدارة الأجهزة)

&#x20; \* billing (الاشتراكات)

&#x20; \* notifications (نظام الإشعارات)

\* في Notification Engine بجدول NotificationEvent

\* النظام فيه Roles + Permissions متقدمة



\---



\## 🎨 قواعد UI/UX (مهم جداً)



\* التصميم نظيف واحترافي (مش dashboard ثقيل)

\* كل العناصر:



&#x20; \* spacing موحد

&#x20; \* ما يتمدد الكرت حسب النص (يعمل scroll داخلي)

\* Sidebar:



&#x20; \* ثابت (fixed)

&#x20; \* بدون scroll

&#x20; \* اللون ممتد للأسفل

&#x20; \* زر logout ثابت تحت

\* دعم كامل عربي + إنجليزي



\---



\## 🔐 قواعد الأمان



\* لا تعطي صلاحيات admin لأي user

\* روابط الإشعارات تكون حسب المستخدم

\* فصل كامل بين admin و subscriber



\---



\## ⚙️ أسلوب العمل المطلوب



\* لا تعيد كتابة ملفات كاملة بدون سبب

\* اعمل تعديلات جزئية (diff-style thinking)

\* اقترح تحسينات لكن لا تكسر النظام

\* وضح لي قبل تنفيذ أي تغيير كبير



\---



\## 📍 الحالة الحالية (حدثها كل مرة)



\[اكتب هنا وين وصلت]

مثال:



\* شغال على تحسين صفحة dashboard

\* في مشكلة بالـ sidebar spacing

\* في bug برسائل الدعم
