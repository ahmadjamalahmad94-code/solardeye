# Heavy v6.2 — Support Mailbox UX

## الهدف
تحويل مركز الدعم من بطاقات كبيرة مناسبة للعرض فقط إلى تجربة Inbox/CRM عملية للرد اليومي، مع إبقاء مركز القيادة كمكان للمراقبة والتحكم.

## التغييرات الرئيسية

### 1. Mailbox Layout جديد
- قائمة دعم مضغوطة بدل البطاقات كافتراضي.
- معاينة محادثة مباشرة في الوسط.
- لوحة خصائص للحالة في الجانب: المسؤول، الحالة، الأولوية، SLA، وإجراءات سريعة.
- دعم التبديل بين عرض القائمة وعرض البطاقات.

### 2. الردود الجاهزة داخل المحادثة
- نقل الردود الجاهزة إلى داخل composer الخاص بكل محادثة.
- زر إدراج الرد داخل textarea.
- زر إدراج الرد مع تغيير الحالة المناسب تلقائيًا.
- زر إرسال مباشر مع رسالة تأكيد قبل الإرسال.

### 3. إجراءات الدعم السريعة
- استلام الحالة.
- انتظار المستخدم.
- إغلاق الحالة.
- إعادة فتح الحالة المغلقة.
- إرسال رد عام أو حفظ ملاحظة داخلية.
- تحديث الحالة والأولوية والمدير المسؤول من نفس composer.

### 4. إصلاح تكدس رسائل النجاح
- دمج الرسائل المتكررة بصريًا.
- عرض حد أقصى لرسالتين toast في نفس الوقت.
- إخفاء تلقائي بعد ثوانٍ قليلة.
- تعطيل أزرار الإرسال بعد أول ضغطة لمنع الإرسال المكرر.

### 5. ترجمة الحالات والأولويات
- open = مفتوح
- waiting_user = بانتظار المستخدم
- resolved = تم الحل
- closed = مغلق
- normal = عادي
- high = مهم
- urgent = عاجل

### 6. تحسين تبويب الدعم داخل ملف المشترك
- تحويل تبويب الدعم في صفحة المشترك الإدارية إلى Mailbox مصغر.
- قائمة مضغوطة بدل بطاقات كبيرة.
- معاينة محادثة وردود جاهزة داخل الرسالة.
- أرشيف مغلق مجمّد بشكل أوضح.

## ما يجب فحصه بعد النشر
1. /admin/support-command-center
2. /admin/users/<id>?lang=ar&tab=support
3. إنشاء رسالة وتذكرة من حساب مشترك.
4. فتح الحالة من مركز القيادة وإرسال رد جاهز عبر:
   - إدراج
   - إدراج + حالة
   - إرسال مباشر
5. تجربة استلام / انتظار المستخدم / إغلاق / إعادة فتح.
6. التأكد أن رسائل النجاح لا تتكدس.
7. التأكد من ظهور الحالات والأولويات بالعربي.
